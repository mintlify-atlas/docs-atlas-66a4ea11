## GET /api/tmdb

Fetch trending movies and TV shows for the current week from The Movie Database (TMDB). This endpoint returns curated content for display on the Connect World platform.

<Note>
  This endpoint fetches trending content from TMDB and returns the top 10 results. The data includes movies, TV shows, and their metadata for display in your application.
</Note>

### Rate Limiting

- **No rate limit** - Public endpoint for content display

### Request

No request body or parameters required. Simple GET request:

```bash
GET /api/tmdb
```

### Response

<ResponseField name="movies" type="TmdbMovie[]">
  Array of trending movies and TV shows (limited to top 10)
  
  Each item contains full metadata from TMDB
</ResponseField>

<ResponseField name="error" type="string" optional>
  Error message if TMDB fetch fails
  
  When error occurs, `movies` will be an empty array and this field contains the error details
</ResponseField>

### TmdbMovie Schema

Each movie/show object in the response contains:

<Expandable title="TmdbMovie Fields">
<ResponseField name="id" type="number">
  TMDB unique identifier for the movie or TV show
  
  Example: `550`
</ResponseField>

<ResponseField name="title" type="string" optional>
  Movie title (only present for movies)
  
  Example: `"Fight Club"`
</ResponseField>

<ResponseField name="name" type="string" optional>
  TV show name (only present for TV shows)
  
  Example: `"Breaking Bad"`
</ResponseField>

<ResponseField name="poster_path" type="string | null">
  Relative path to poster image
  
  Prepend with `https://image.tmdb.org/t/p/w500` for full URL
  
  Example: `"/pB8BM7pdSp6B6Ih7QZ4DrQ3PmJK.jpg"`
</ResponseField>

<ResponseField name="backdrop_path" type="string | null">
  Relative path to backdrop image
  
  Prepend with `https://image.tmdb.org/t/p/original` for full URL
  
  Example: `"/fCayJrkfRaCRCTh8GqN30f8oyQF.jpg"`
</ResponseField>

<ResponseField name="overview" type="string">
  Plot summary or description
  
  Example: `"A ticking-time-bomb insomniac and a slippery soap salesman..."`
</ResponseField>

<ResponseField name="vote_average" type="number">
  Average rating from TMDB users (0-10 scale)
  
  Example: `8.433`
</ResponseField>

<ResponseField name="release_date" type="string" optional>
  Movie release date (ISO 8601 format, only for movies)
  
  Example: `"1999-10-15"`
</ResponseField>

<ResponseField name="first_air_date" type="string" optional>
  TV show first air date (ISO 8601 format, only for TV shows)
  
  Example: `"2008-01-20"`
</ResponseField>

<ResponseField name="genre_ids" type="number[]">
  Array of genre IDs from TMDB
  
  Example: `[18, 53, 35]` (Drama, Thriller, Comedy)
</ResponseField>

<ResponseField name="media_type" type="string" optional>
  Type of media
  
  Values: `"movie"` or `"tv"`
</ResponseField>
</Expandable>

```typescript TypeScript
interface TmdbMovie {
  id: number;
  title?: string;
  name?: string;
  poster_path: string | null;
  backdrop_path: string | null;
  overview: string;
  vote_average: number;
  release_date?: string;
  first_air_date?: string;
  genre_ids: number[];
  media_type?: string;
}

interface TmdbResponse {
  movies: TmdbMovie[];
  error?: string;
}

const fetchTrending = async (): Promise<TmdbMovie[]> => {
  const response = await fetch('https://your-domain.com/api/tmdb');
  const data: TmdbResponse = await response.json();
  return data.movies;
};
```

### Error Response

<Expandable title="500 Internal Server Error">
```json
{
  "movies": [],
  "error": "Failed to fetch TMDB data"
}
```

Returned when:
- TMDB API is unavailable
- Invalid TMDB API key
- Network error
- Rate limit exceeded on TMDB side

The endpoint returns status 500 but still includes an empty `movies` array for graceful degradation.
</Expandable>

## Image URLs

TMDB returns relative paths for images. Use the helper function to generate full URLs:

```typescript
// From tmdbService.ts:44-47
export function getPosterUrl(path: string | null, size = 'w500'): string {
  if (!path) return '/images/placeholder.jpg';
  return `https://image.tmdb.org/t/p/${size}${path}`;
}
```

### Available Image Sizes

**Poster sizes:**
- `w92` - Thumbnail
- `w154` - Small
- `w185` - Medium (default mobile)
- `w342` - Large
- `w500` - Extra large (recommended)
- `w780` - XXL
- `original` - Full resolution

**Backdrop sizes:**
- `w300` - Small
- `w780` - Medium
- `w1280` - Large
- `original` - Full resolution (recommended)

### Example: Rendering Images

```jsx
function MovieCard({ movie }) {
  const posterUrl = movie.poster_path
    ? `https://image.tmdb.org/t/p/w500${movie.poster_path}`
    : '/images/placeholder.jpg';
  
  const backdropUrl = movie.backdrop_path
    ? `https://image.tmdb.org/t/p/original${movie.backdrop_path}`
    : null;
  
  const title = movie.title || movie.name;
  const date = movie.release_date || movie.first_air_date;
  
  return (
    <div className="movie-card">
      <img src={posterUrl} alt={title} />
      <h3>{title}</h3>
      <p>Rating: {movie.vote_average}/10</p>
      <p>Release: {date}</p>
      <p>{movie.overview}</p>
    </div>
  );
}
```

## TMDB Service

The endpoint uses the `getTrendingAll` function from the TMDB service:

```typescript
// From tmdbService.ts:29-32
export async function getTrendingAll(timeWindow: 'day' | 'week' = 'week'): Promise<TmdbMovie[]> {
  const { data } = await tmdbClient.get<TmdbResponse>(`/trending/all/${timeWindow}`);
  return data.results.slice(0, 10);
}
```

### Service Configuration

<Note>
  The TMDB service is configured with:
  
  - **Base URL:** `https://api.themoviedb.org/3`
  - **Language:** `es-ES` (Spanish)
  - **Authentication:** API key from environment variable
  - **Default time window:** `week`
  - **Result limit:** Top 10 items
</Note>

## Use Cases

### Homepage Hero Section

Display trending content on your homepage:

```jsx
import { useEffect, useState } from 'react';

function HeroSection() {
  const [trending, setTrending] = useState([]);
  
  useEffect(() => {
    fetch('/api/tmdb')
      .then(res => res.json())
      .then(data => setTrending(data.movies.slice(0, 5)));
  }, []);
  
  return (
    <div className="hero">
      {trending.map(item => (
        <div key={item.id} className="hero-slide">
          <img 
            src={`https://image.tmdb.org/t/p/original${item.backdrop_path}`}
            alt={item.title || item.name}
          />
        </div>
      ))}
    </div>
  );
}
```

### Content Catalog

Build a browsable catalog of trending content:

```jsx
function TrendingCatalog() {
  const [movies, setMovies] = useState([]);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    fetch('/api/tmdb')
      .then(res => res.json())
      .then(data => {
        setMovies(data.movies);
        setLoading(false);
      })
      .catch(err => {
        console.error('Failed to load trending:', err);
        setLoading(false);
      });
  }, []);
  
  if (loading) return <div>Loading trending content...</div>;
  
  return (
    <div className="catalog">
      <h2>Trending This Week</h2>
      <div className="grid">
        {movies.map(movie => (
          <MovieCard key={movie.id} movie={movie} />
        ))}
      </div>
    </div>
  );
}
```

## Caching Recommendations

<Note>
  Consider implementing caching for TMDB data:
  
  - **Client-side:** Cache in localStorage or React Query
  - **Server-side:** Cache with Redis or in-memory cache
  - **TTL:** 6-12 hours (trending data changes slowly)
  - **Benefit:** Reduce TMDB API calls and improve response time
</Note>
